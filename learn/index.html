Pondiverse - How to take part in the Pondiverse

Hello!

Maybe you've seen the cool things people are doing and want to join in? Or perhaps you want to take part by adding your own tool to the Pondiverse?

This guide will show you how to do both. We'll start with using other people's tools to take part, and then we'll see how to add your own tool to the Pondiverse.

Using other people's tools

  1. Find a tool that's part of the Pondiverse. There is a list of tools on the create page.
  2. Click on a tool to go to its website.
  3. Make your thing in the tool. Keep fiddling until you're happy. It's like arranging flowers in a vase. This is going to be your creation.
  4. Click the blue button in the bottom right corner. It looks like this:
  5. Add a title and click submit.

Congratulations, you are now participating in the Pondiverse!

Adding your tool to the Pondiverse

  1. Import the Pondiverse script from this website
  2. Write a function that returns the data you want to be shown in the gallery
  3. Call the addPondiverseButton() function somewhere after that

You can import the Pondiverse script from this website like so: 

import {addPondiverseButton} from 'https://www.pondiverse.com/script/pondiverse.js'
Put it somewhere near the top your script, before you do the rest.

Then, write a function that returns the data Pondiverse wants. It should return a javascript object with the following attributes:

  • type: The name of your tool
  • data: An arbitrary data attribute, that will in future be used to link back to or embed your creation.
  • image: The creation itself, encoded as a dataURL. This is what is rendered to the gallery.
You get to choose what these attributes are. You can make them anything you like, but images are popular. We trust you.

Make this function an attribute of the global window object called getPondiverseCreation so that the Pondiverse script can access it. My code for pNMRsimJS looks like this: 

window.getPondiverseCreation = () => {
    return {
        type: "pNMRsimJS",
        data: JSON.stringify(simulation.pulseSequence),
        image: getScreenshot(),
    };
};
I've set my data attribute to be the pulse sequence that produced the simulation, and I've set my image attribute to be the result of a function that renders and then returns the data on my canvas.

If you're using THREE.js, my getScreenshot() function might be useful. It looks like this: 

function getScreenshot() {
    render();
    return canvas?.toDataURL("image/png");
}
With THREE.js, I need to render (with renderer.render(scene, camera)) my canvas one last time before getting its data. I do this along with some other bits in my render() function.

If you aren't using THREE.js, then you can have image: canvas?.toDataURL("image/png"), in your getPondiverseCreation function. This line checks if the canvas exists, and if it does (the ? operator), calls the toDataURL function on it to convert it to a png image.

Finally, add the line 

addPondiverseButton();
anywhere in your script, after this function you just wrote. This is what adds the Pondiverse button to your page.

Congratulations, you are now participating in the Pondiverse!

That's it! Have a nice day :)

Secret bonus section: adding your tool to this website

If you are happy using github, you can make a pull request to the participants.js file. This contains one big object with the names of all the tools, along with links to their websites. This list gets put on the create page and with the titles in the gallery.

Say, I'd just added a tool called "splat" to the Pondiverse. I'd go to the github and click on the participants.js page in the file explorer. Then I'd click on the little pen in the top-right-hand corner of the code listing. Then, I'd click the "Fork Repository" button and add a line like the following: 

const participants = {
      "screenpond": {name: "Screenpond", url: "https://screenpond.cool/"},
      "chaoskit": {name: "Chaoskit", url: "https://evolved.systems/chaoskit/"},
      "pnmrsimjs": {name: "pNMRsimJS", url: "https://mags.omg.lol/simulator.php"},
++++  "splat": {name: "Splat!", url: "https://example.org"},
    };
(but without the +s). name is the pretty display name, and url is the url of the tool. Then I'd click "Commit Changes" and then "Make Pull Request". I'd type some nonsense in the title or whatever and then I'd click submit, and someone will later merge it in :)

If you're not happy using github you can send me message on the fediverse (mags@mags.omg.lol) and I'll do it for you.
